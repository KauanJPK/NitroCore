# Why NitroCore?

- The name was derived from the nickname for nitrous oxide (N₂O). Used in racing, this gas is injected into the engine
to sharply boost the vehicle's power and speed—representing NitroCore as a powerful optimization core.
- NitroCore was created to solve a specific problem: excessive CPU usage in Minecraft. It is designed to improve
performance on everything from low-end devices to high-end gaming computers, providing smoother gameplay in large 
modpacks and modern Minecraft versions. Support for newer Minecraft versions will begin only after NitroCore reaches a
stable release for Minecraft Forge 1.7.10.

# What we believe

- NitroCore is a project in constant development. Great software is not built from thousands of random lines of code. 
It is built one meaningful commit at a time.
- NitroCore is not just another FPS mod; NitroCore focuses on eliminating unnecessary CPU work
caused by particles, animations, rendering and excessive memory allocations.
- NitroCore will always remain free and open source.
- Open source makes software better.
- Simplicity is better than unnecessary complexity.
- Users should always stay in control of their experience.

---

# Our Community

- Everyone is welcome to contribute. So if you want to help, we welcome ideas, bug reports, compatibility reports and Pull Requests.
Every contribution helps make NitroCore better.

---

# Our Promise

- We will always prioritize compatibility.
- We will always prioritize measurable optimizations.
- We will never sacrifice quality for release dates.
- We will always keep NitroCore open source.

---

# Manifesto Rules

- NitroCore is an **open-source** project, so if you see someone copying or redistributing the project in violation of its license
and attempting to distribute it as paid software, please report copyright violations through the official repository.
- Every Pull Request should have a clear purpose and improve the project.
- NitroCore is designed to remain lightweight. Every dependency must justify its existence, and every optimization
should reduce overhead rather than introduce unnecessary complexity.
- NitroCore will never become closed source.
- NitroCore will never lock features behind payments.
- The project will always remain accessible to everyone.
